## Use-Case Coverage

Every known use case served by the above projects can be built on `ovpn-mgmt-codec` as the protocol layer.

| Use case                      | What it needs                                                            | ovpn-mgmt-codec types                                                                                                              | Projects that do only this                                                                                                                                                                                                                                                                                                               |
| ----------------------------- | ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Metrics / monitoring**      | `status`, `load-stats`, `version`, `pid`                                 | `OvpnCommand::Status`, `LoadStats`, `Version`, `Pid`                                                                               | [openvpn-management](https://github.com/tmorgansl/openvpn-management), [openvpn-api](https://github.com/Jamie-/openvpn-api), [go-openvpn](https://github.com/pavel-d/go-openvpn), [openvpn-monitor](https://github.com/furlongm/openvpn-monitor), [Netdata collector](https://pkg.go.dev/github.com/netdata/go.d.plugin/modules/openvpn) |
| **Auth plugins**              | `>CLIENT:` events, `client-auth`/`client-deny`, `hold release`           | `Notification::Client` (with ENV map), `OvpnCommand::ClientAuth`, `ClientAuthNt`, `ClientDeny`, `ClientPendingAuth`, `HoldRelease` | [openvpn-auth-oauth2](https://github.com/jkroepke/openvpn-auth-oauth2), [openvpn-auth-azure-ad](https://github.com/jkroepke/openvpn-auth-azure-ad)                                                                                                                                                                                       |
| **GUIs / process wrappers**   | `signal`, `>STATE:`, `>LOG:`, `state`                                    | `OvpnCommand::Signal`, `State`, `StateStream`, `Log`; `Notification::State`, `Notification::Log`                                   | [node-openvpn](https://github.com/luigiplr/node-openvpn), [go-openvpn](https://github.com/mysteriumnetwork/go-openvpn), [go-openvpn](https://github.com/stamp/go-openvpn), [gopenvpn](https://github.com/NordSecurity/gopenvpn)                                                                                                          |
| **Connection kill**           | `status` + `kill`                                                        | `OvpnCommand::Status`, `OvpnCommand::Kill(KillTarget::CommonName\|Address)`                                                        | [php-openvpn-connection-manager](https://github.com/letsconnectvpn/php-openvpn-connection-manager)                                                                                                                                                                                                                                       |
| **Password / credential UIs** | `>PASSWORD:` prompts, `username`/`password` commands, challenge-response | `PasswordNotification` (all 6 subtypes), `OvpnCommand::Username`, `Password`, `ChallengeResponse`, `StaticChallengeResponse`       | [openvpn-mgmt-go][cmd-cedws] has username/password/challenge commands; no other library types `>PASSWORD:` subtypes                                                                                                                                                                                                                      |
| **Hardware token (PKCS#11)**  | Token enumeration, certificate selection                                 | `OvpnCommand::Pkcs11IdCount`, `Pkcs11IdGet`; `Notification::Pkcs11IdCount`, `OvpnMessage::Pkcs11IdEntry`                           | [openvpn-mgmt-go][cmd-cedws] also has these commands                                                                                                                                                                                                                                                                                     |
| **External key / signing**    | Sign data with an external HSM or agent (e.g. ssh-agent, PKCS#11)        | `Notification::RsaSign`, `OvpnCommand::RsaSig`, `OvpnCommand::Certificate`                                                         | [openvpn-mgmt-go][cmd-cedws] also has these commands                                                                                                                                                                                                                                                                                     |
| **Interactive prompts**       | Token insertion confirmations, user-provided strings                     | `Notification::NeedOk`, `Notification::NeedStr`; `OvpnCommand::NeedOk`, `NeedStr`                                                  | [openvpn-mgmt-go][cmd-cedws] also has these commands                                                                                                                                                                                                                                                                                     |
| **Remote / proxy override**   | Dynamic server or proxy selection at connect time                        | `Notification::Remote`, `Notification::Proxy`; `OvpnCommand::Remote(RemoteAction)`, `Proxy(ProxyAction)`                           | [openvpn-mgmt-go][cmd-cedws] also has these commands                                                                                                                                                                                                                                                                                     |
| **Bandwidth tracking**        | Per-client and per-tunnel byte counters                                  | `OvpnCommand::ByteCount`; `Notification::ByteCount`, `Notification::ByteCountCli`                                                  | [gopenvpn][evt-gopenvpn] and [go-openvpn-mgmt][evt-apparentlymart] also parse BYTECOUNT events; [openvpn-mgmt-go][cmd-cedws] has the command                                                                                                                                                                                             |
| **Verbosity / mute tuning**   | Runtime log-level changes without restart                                | `OvpnCommand::Verb`, `Mute`                                                                                                        | [openvpn-mgmt-go][cmd-cedws] also has these commands                                                                                                                                                                                                                                                                                     |
| **Management interface auth** | Password-protected management sockets                                    | `OvpnMessage::PasswordPrompt`, `OvpnCommand::ManagementPassword`                                                                   | No other library models this as a typed exchange                                                                                                                                                                                                                                                                                         |
